実務でありそうな「大量のログファイルを急いで解析してほしい」という状況を題材に、Goの並行処理を学ぶハンズオンです。参加者はペアで、最初は遅い逐次処理を書き、段階的に並行処理へと改良していきます。マシンスペックの差が出ないよう「改善率」で競い、各段階で工夫を共有し合います。

以下のような流れを想定しています。

- Phase0: 講義パート（35分）
  - Go基礎とログファイルの読み方
  - goroutineとchannelの基本
  - 今日のルール説明
- Phase 1: 逐次処理版実装（15分）
- Phase 2: 並行処理版実装（20分）
- Phase 3: 自由最適化（15分）
- Phase 4: 結果発表（10分）

## ワークショップから学べること

- 並行処理の実践パターン
goroutineとchannelの基本から、実務でよく使うパターンまで身につけることができます。
- 段階的な改善アプローチ
まず動くものを作り、次に並行化し、最後に細かく最適化する。この3段階のプロセスは実務のパフォーマンスチューニングでそのまま使えます。
- ペアプログラミングの経験
2人1組で90分間程度ずっと相談しながら進めます。一人で黙々とやるより楽しいし、説明することで理解も深まります。
- すぐ使える実装パターン
簡単なログ解析だけでなく、大量ファイルの処理やバッチ処理など、似たような課題にすぐ応用できる手法を身につけられます。
